# Channels & Gateway (Spec 2) — Docker Provisioning + CLI Integration

## Overview

Wire the gateway into the deployment pipeline. The Docker provider provisions gateway containers, the CLI generates a routing config file after deploying agents, and the gateway loads routes from this file on startup.

## Decisions

| Decision | Choice |
|----------|--------|
| Gateway image | Built from agentstack-gateway source, same pattern as agent images |
| Route config | File-based (`routes.json`), mounted into gateway container |
| CLI integration | No new commands — gateway provisioned automatically by `apply` |
| Route updates | CLI regenerates `routes.json`, restarts gateway |
| Destroy | Deregisters routes; `--include-resources` removes gateway containers |

## Gateway Container

**Image build:** The Docker provider writes gateway source files to `.agentstack/gateway-{name}/`, generates a Dockerfile, and builds the image.

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "server.py"]
```

The build context contains the gateway server code (server.py, router.py, providers/) and a requirements.txt.

**Container config:**
- Name: `agentstack-gateway-{gateway.name}`
- Network: `agentstack-net`
- Port: gateway config port (default 8080), exposed on host
- Volume: `.agentstack/gateway-{name}/routes.json` mounted as `/app/routes.json`
- Labels: `agentstack.gateway={name}`
- Environment: secrets from channel provider configs (bot tokens, app tokens)

## Routes File

**Location:** `.agentstack/gateway-{name}/routes.json`

**Format:**
```json
{
  "providers": [
    {
      "name": "internal-slack",
      "type": "slack",
      "config": {
        "bot_token": "xoxb-...",
        "app_token": "xapp-..."
      }
    }
  ],
  "routes": [
    {
      "provider_name": "internal-slack",
      "agent_name": "support-bot",
      "agent_url": "http://agentstack-support-bot:8000",
      "channels": ["#support", "#help"],
      "listen": "mentions",
      "threads": true,
      "dm": true
    }
  ]
}
```

**Generated by:** CLI `apply` command after deploying agents.

**Loaded by:** Gateway server on startup — reads `/app/routes.json`, registers providers and routes.

**Updated by:** CLI regenerates the file and restarts the gateway container when agents are added/removed.

## Updated Apply Flow

```
agentstack apply
  1. Load agent definition
  2. Validate via adapter
  3. Generate agent code
  4. Provision resources (Postgres, SQLite) — existing
  5. Scan agent.channels for SlackChannel instances
  6. For each unique gateway referenced by channels:
     a. Build gateway image if not exists
     b. Collect channel providers and routes
     c. Resolve secrets (bot tokens from env)
     d. Write routes.json to .agentstack/gateway-{name}/
     e. Start/restart gateway container with routes.json mounted
  7. Deploy agent container — existing
```

## Updated Destroy Flow

```
agentstack destroy
  1. Remove agent container — existing
  2. Read agent channels, find gateway references
  3. Remove agent's routes from routes.json
  4. Restart gateway container with updated routes.json

agentstack destroy --include-resources
  1. Remove agent container
  2. Remove resource containers (Postgres, etc.) — existing
  3. Remove gateway containers
```

### packages/python/agentstack-gateway/

**Modified: `server.py`**

- On startup, load `/app/routes.json` if it exists
- Parse providers and routes, register them into the router
- Start channel provider runners for each provider

Add a `load_routes_file(path)` function:
```python
def load_routes_file(path: str = "/app/routes.json"):
    """Load providers and routes from a JSON config file."""
```

Add a lifespan handler that loads routes on startup:
```python
@asynccontextmanager
async def lifespan(app):
    load_routes_file()
    # Start all registered providers
    for runner in providers.values():
        asyncio.create_task(runner.start())
    yield
    # Stop all providers
    for runner in providers.values():
        await runner.stop()
```

### packages/python/agentstack-cli/

**Modified: `commands/apply.py`**

- After deploying agent, scan channels for SlackChannel
- Collect gateway info, write routes.json
- Call provider to provision/restart gateway

**Modified: `commands/destroy.py`**

- After removing agent, update routes.json to remove agent's routes
- Restart gateway if routes changed

## What This Spec Does NOT Cover

- Gateway scaling/replication
- Hot-reload of routes (gateway restart required for now)
- Authentication on gateway management API
- Discord, Telegram, or other channel providers
- Multiple agents in a single `apply` (currently one agent per file)
