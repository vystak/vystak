Separated Provider — Azure as cloud, services as targets

The provider is Azure. Container Apps and Postgres are independent choices on that provider.

A

Provider + Platform + Resource as separate concepts

Azure is the provider. Container Apps is the platform. Postgres is a resource. Each references the provider independently.

Python
import agentstack as ast

# 1. Declare the cloud provider
azure = ast.Provider(
    name="azure",
    type="azure",
    config={
        "location": "eastus2",
        "resource_group": "agentstack-prod",  # optional
    },
)

# 2. Model
anthropic = ast.Provider(name="anthropic", type="anthropic")
model = ast.Model(name="claude", provider=anthropic, model_name="claude-sonnet-4-20250514")

# 3. Agent — platform and resources reference the azure provider
agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(name="aca", type="container-apps", provider=azure),
    resources=[
        ast.Resource(name="sessions", engine="postgres", provider=azure),
    ],
    channels=[ast.Channel(name="api", type=ast.ChannelType.API)],
)
YAML
name: support-bot
model:
  name: claude
  provider: { name: anthropic, type: anthropic }
  model_name: claude-sonnet-4-20250514
platform:
  name: aca
  type: container-apps
  provider: { name: azure, type: azure }
resources:
  - name: sessions
    engine: postgres
    provider: { name: azure, type: azure }
channels:
  - name: api
    type: api
B

Same but with bring-your-own resource

Skip provisioning for a resource by providing a connection string. Provider is still Azure for the platform.

Python — existing Postgres
agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(name="aca", type="container-apps", provider=azure),
    resources=[
        # Bring your own — no provider, just a connection string
        ast.Resource(
            name="sessions",
            engine="postgres",
            config={"connection_string_env": "DATABASE_URL"},
        ),
    ],
)
C

Mix providers — Azure compute, Docker Postgres

Platform on Azure, but resource on a different provider. Each is independent.

Python — mixed providers
azure = ast.Provider(name="azure", type="azure", config={"location": "eastus2"})
docker = ast.Provider(name="docker", type="docker")

agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(name="aca", type="container-apps", provider=azure),
    resources=[
        # Postgres runs locally in Docker
        ast.Resource(name="sessions", engine="postgres", provider=docker),
    ],
)
