What should replace ast.Resource?

Resource is too generic. What better captures the intent?

A

Typed resource classes

Each resource type is its own class. Explicit, discoverable, IDE-friendly.

Python
agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(type="container-apps", provider=azure),
    session_store=ast.PostgresStore(provider=azure),
    memory_store=ast.PostgresStore(
        provider=azure,
        config={"sku": "Standard_B1ms"},
    ),
)

# Or bring your own:
agent = ast.Agent(
    ...
    session_store=ast.PostgresStore(
        connection_string_env="DATABASE_URL",
    ),
)

Clear what each thing is. No engine="postgres" string. Type-safe config per store type.

B

Named fields on Agent, generic Store

Agent has explicit fields for each concern. One Store class handles backing config.

Python
agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(type="container-apps", provider=azure),
    sessions=ast.Store(engine="postgres", provider=azure),
    memory=ast.Store(engine="postgres", provider=azure, config={
        "sku": "Standard_B1ms",
    }),
)

# Bring your own:
agent = ast.Agent(
    ...
    sessions=ast.Store(
        engine="postgres",
        connection_string_env="DATABASE_URL",
    ),
)

Agent shape is clear (sessions, memory are named fields). Store is flexible but still generic.

C

Keep Resource but scope it to services

Rename to Service. Resources list infra services the agent depends on.

Python
agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(type="container-apps", provider=azure),
    services=[
        ast.Postgres(name="sessions", provider=azure),
        ast.Postgres(name="memory", provider=azure, config={
            "sku": "Standard_B1ms",
        }),
        ast.Redis(name="cache", provider=azure),
    ],
)

# Bring your own:
agent = ast.Agent(
    ...
    services=[
        ast.Postgres(name="sessions",
            connection_string_env="DATABASE_URL",
        ),
    ],
)

Extensible list. Each service is a typed class. Easy to add Redis, Qdrant, etc. later.

D

Hybrid — named fields + services list

Common concerns (sessions, memory) are named fields. Extra infra goes in services.

Python
agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(type="container-apps", provider=azure),
    # First-class agent concerns
    sessions=ast.Postgres(provider=azure),
    memory=ast.Postgres(provider=azure),
    # Additional services the agent needs
    services=[
        ast.Redis(name="cache", provider=azure),
        ast.Qdrant(name="vectors", provider=azure),
    ],
)

# Bring your own:
agent = ast.Agent(
    ...
    sessions=ast.Postgres(connection_string_env="DATABASE_URL"),
    memory=ast.Postgres(connection_string_env="MEMORY_URL"),
)

Best of both: sessions/memory are obvious, but you can still attach arbitrary services.
