How should agents discover each other on Azure?

Today on Docker, agents use container names on a shared network. What's the Azure equivalent?

A

Shared Container App Environment

All agents in the same workspace deploy into one ACA Environment. They discover each other via internal DNS names (like Docker network). No public exposure needed for agent-to-agent calls. Closest to how Docker works today.

Pros

  • Simple DNS-based discovery
  • No public endpoints for internal traffic
  • Minimal config change from Docker

Cons

  • All agents share scaling/region
  • One environment per workspace
B

VNet + Private DNS Zone

Each agent gets its own ACA Environment but they share a VNet with private DNS. More isolated but more infrastructure to manage.

Pros

  • Independent scaling per agent
  • Better isolation

Cons

  • More resources to provision
  • Slower deploys
  • More complex networking
C

Public HTTPS + FQDN

Each agent gets an external HTTPS URL. Agents call each other via public endpoints. Simplest to set up but traffic goes over the internet.

Pros

  • Simplest to provision
  • Works across regions/subscriptions

Cons

  • Traffic over public internet
  • Need auth between agents
  • Latency overhead
