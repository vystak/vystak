Azure Provider — Code Declaration Options

How much Azure config lives in the agent definition vs environment?

A

Minimal — Environment-Driven

YAML just declares the provider type. Everything else from az login context or env vars.

Python
azure = ast.Provider(name="azure", type="azure-container-apps")

agent = ast.Agent(
    name="support-bot",
    model=model,
    channels=[ast.Channel(name="api", type=ast.ChannelType.API)],
    resources=[ast.Resource(name="sessions", provider=azure, engine="postgres")],
)
# Region, subscription, RG all come from:
#   az login context, or
#   AZURE_SUBSCRIPTION_ID, AZURE_LOCATION, AZURE_RESOURCE_GROUP
YAML
name: support-bot
resources:
  - name: sessions
    provider:
      name: azure
      type: azure-container-apps
    engine: postgres
B

Explicit — Fully Self-Describing

Everything in the definition. Portable across machines but tied to one Azure account.

Python
azure = ast.Provider(
    name="azure",
    type="azure-container-apps",
    config={
        "subscription_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
        "location": "eastus2",
        "resource_group": "agentstack-prod",
        "container_registry": "myregistry.azurecr.io",
        "sku": "Consumption",
    },
)

agent = ast.Agent(
    name="support-bot",
    model=model,
    resources=[ast.Resource(name="sessions", provider=azure, engine="postgres")],
)
C

Layered — Logical Intent in Code, Credentials from Env

Definition says where and how, but not who. Credentials always from environment.

Python
azure = ast.Provider(
    name="azure",
    type="azure-container-apps",
    config={
        "location": "eastus2",
        "resource_group": "agentstack-prod",  # optional, auto-generated if omitted
    },
)

agent = ast.Agent(
    name="support-bot",
    model=model,
    resources=[
        ast.Resource(name="sessions", provider=azure, engine="postgres"),
        # OR bring your own:
        ast.Resource(
            name="sessions",
            provider=azure,
            engine="postgres",
            config={"connection_string_env": "DATABASE_URL"},  # skip provisioning
        ),
    ],
)
YAML
name: support-bot
resources:
  - name: sessions
    provider:
      name: azure
      type: azure-container-apps
      config:
        location: eastus2
        resource_group: agentstack-prod
    engine: postgres

# Or bring your own Postgres:
  - name: sessions
    provider:
      name: azure
      type: azure-container-apps
    engine: postgres
    config:
      connection_string_env: DATABASE_URL
