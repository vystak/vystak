Pulumi vs AgentStack — Azure Container Apps + Postgres

How Pulumi declares the same infrastructure, and what we can learn from it.

Pulumi (Python) — ~60 lines
import pulumi
import pulumi_azure_native as azure

# Provider is implicit — from `az login` or config
# But can be explicit:
provider = azure.Provider("azure", location="eastus2")

rg = azure.resources.ResourceGroup("rg",
    resource_group_name="agentstack-prod",
    opts=pulumi.ResourceOptions(provider=provider),
)

logs = azure.operationalinsights.Workspace("logs",
    resource_group_name=rg.name,
    sku=azure.operationalinsights.WorkspaceSkuArgs(name="PerGB2018"),
)

env = azure.app.ManagedEnvironment("env",
    resource_group_name=rg.name,
    app_logs_configuration=azure.app.AppLogsConfigurationArgs(
        destination="log-analytics",
        log_analytics_configuration=azure.app.LogAnalyticsConfigurationArgs(
            customer_id=logs.customer_id,
            shared_key=logs...
        ),
    ),
)

acr = azure.containerregistry.Registry("acr",
    resource_group_name=rg.name,
    sku=azure.containerregistry.SkuArgs(name="Basic"),
    admin_user_enabled=True,
)

postgres = azure.dbforpostgresql.Server("pg",
    resource_group_name=rg.name,
    sku=azure.dbforpostgresql.SkuArgs(
        name="Standard_B1ms", tier="Burstable",
    ),
    version="16",
    storage=azure.dbforpostgresql.StorageArgs(
        storage_size_gb=32,
    ),
    administrator_login="agentstack",
    administrator_login_password=secret,
)

db = azure.dbforpostgresql.Database("db",
    resource_group_name=rg.name,
    server_name=postgres.name,
    database_name="agent_sessions",
)

app = azure.app.ContainerApp("support-bot",
    resource_group_name=rg.name,
    managed_environment_id=env.id,
    configuration=azure.app.ConfigurationArgs(
        ingress=azure.app.IngressArgs(
            external=True, target_port=8000,
        ),
        secrets=[...],
        registries=[...],
    ),
    template=azure.app.TemplateArgs(
        containers=[azure.app.ContainerArgs(
            name="support-bot",
            image=f"{acr.login_server}/support-bot:latest",
            resources=azure.app.ContainerResourcesArgs(
                cpu=0.5, memory="1Gi",
            ),
            env=[
                azure.app.EnvironmentVarArgs(
                    name="DATABASE_URL",
                    secret_ref="db-url",
                ),
            ],
        )],
    ),
)
AgentStack (Python) — ~15 lines
import agentstack as ast

azure = ast.Provider(
    name="azure",
    type="azure",
    config={"location": "eastus2"},
)

anthropic = ast.Provider(
    name="anthropic",
    type="anthropic",
)

model = ast.Model(
    name="claude",
    provider=anthropic,
    model_name="claude-sonnet-4-20250514",
)

agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(
        name="aca",
        type="container-apps",
        provider=azure,
    ),
    resources=[
        ast.Resource(
            name="sessions",
            engine="postgres",
            provider=azure,
        ),
    ],
    channels=[
        ast.Channel(name="api", type=ast.ChannelType.API),
    ],
    secrets=[
        ast.Secret(name="ANTHROPIC_API_KEY"),
    ],
)

Key Patterns from Pulumi

Provider is a first-class object

Pulumi passes provider to each resource. AgentStack does the same — azure is referenced by both platform and resources independently.

Implicit vs explicit config

Pulumi tries az login first, falls back to explicit config. AgentStack should do the same — provider config is optional overrides on top of CLI context.

Resources are wired by reference

Pulumi passes rg.name, env.id between resources. AgentStack hides this — the provider knows how to wire RG → ACR → ACA → Postgres internally.

AgentStack's job is the 45-line gap

Pulumi: ~60 lines of infra code. AgentStack: ~15 lines of intent. The provider plugin fills in the 45 lines of Azure-specific wiring automatically.
