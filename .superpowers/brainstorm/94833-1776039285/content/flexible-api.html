Flexible API — Pulumi-level control, agent-level focus

Progressive complexity: simple defaults, full overrides when you need them.

A

Level 1 — Minimal (today's experience)

Just declare intent. Provider fills in all defaults.

Python — 10 lines
azure = ast.Provider(name="azure", type="azure")

agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(type="container-apps", provider=azure),
    resources=[ast.Resource(name="sessions", engine="postgres", provider=azure)],
)

Provider auto-detects subscription, picks cheapest SKU, generates RG name, wires everything.

B

Level 2 — Configure provider & platform

Control where and how things run. Resources still use smart defaults.

Python
azure = ast.Provider(
    name="azure",
    type="azure",
    config={
        "subscription_id": "xxxx-xxxx",
        "location": "eastus2",
        "resource_group": "my-agents-rg",
        "tags": {"team": "platform", "env": "prod"},
    },
)

agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(
        type="container-apps",
        provider=azure,
        config={
            "cpu": 1.0,
            "memory": "2Gi",
            "min_replicas": 1,
            "max_replicas": 5,
            "ingress": "external",
        },
    ),
    resources=[ast.Resource(name="sessions", engine="postgres", provider=azure)],
)
C

Level 3 — Full resource control

Override every resource. Mix provisioned and existing. Pulumi-level control.

Python
azure = ast.Provider(
    name="azure",
    type="azure",
    config={
        "location": "eastus2",
        "resource_group": "my-agents-rg",
        "tags": {"team": "platform"},
    },
)

agent = ast.Agent(
    name="support-bot",
    model=model,
    platform=ast.Platform(
        type="container-apps",
        provider=azure,
        config={
            "environment": "my-existing-env",   # use existing ACA env
            "registry": "myacr.azurecr.io",     # use existing ACR
            "cpu": 0.5,
            "memory": "1Gi",
            "min_replicas": 0,                  # scale to zero
            "max_replicas": 10,
            "ingress": "internal",              # no public endpoint
        },
    ),
    resources=[
        # Provision new Postgres with specific config
        ast.Resource(
            name="sessions",
            engine="postgres",
            provider=azure,
            config={
                "sku": "Standard_B1ms",
                "storage_gb": 64,
                "version": "16",
                "high_availability": False,
            },
        ),
        # Bring your own — skip provisioning entirely
        ast.Resource(
            name="memory",
            engine="postgres",
            config={"connection_string_env": "MEMORY_DATABASE_URL"},
        ),
    ],
    secrets=[
        ast.Secret(name="ANTHROPIC_API_KEY"),
        ast.Secret(name="MEMORY_DATABASE_URL"),  # injected from Key Vault or env
    ],
)
D

Level 4 — Multi-provider fleet

Multiple Azure providers for different regions/subscriptions. Independent wiring.

Python
azure_east = ast.Provider(
    name="azure-east", type="azure",
    config={"location": "eastus2", "resource_group": "agents-east"},
)
azure_west = ast.Provider(
    name="azure-west", type="azure",
    config={"location": "westus2", "resource_group": "agents-west"},
)

# Frontend agent in east
frontend = ast.Agent(
    name="frontend-bot",
    model=model,
    platform=ast.Platform(type="container-apps", provider=azure_east),
    resources=[ast.Resource(name="sessions", engine="postgres", provider=azure_east)],
)

# Backend agent in west
backend = ast.Agent(
    name="backend-bot",
    model=model,
    platform=ast.Platform(type="container-apps", provider=azure_west),
    resources=[ast.Resource(name="sessions", engine="postgres", provider=azure_west)],
)
